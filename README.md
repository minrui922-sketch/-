# 闪念课堂助手 (Flash Thought Classroom Assistant)

> 智能学习伴侣 · 专注思考的工具 | Beta

基于**微信小程序 + 云开发**的 AI 学习助手，提供课堂图片摘要与代码报错诊断两大核心能力，帮助你快速提炼知识和定位 Bug。

---

## 功能概览

### 📷 课堂助手 — AI 图像摘要

拍摄或上传课堂 PPT、文献图片 → 上传至微信云存储 → 调用视觉大模型（Qwen-VL-Max）分析图像内容 → 返回精简的核心摘要（Markdown 格式，200 字以内）。

**操作流程：**
1. 点击上传区域，拍照或从相册选择
2. 图片自动上传至云存储
3. 点击「生成核心摘要」按钮，等待 AI 分析
4. 查看渲染后的 Markdown 摘要结果

### 🐞 代码诊断 — 报错一键分析

粘贴终端报错、编译错误或异常堆栈信息 → 调用大模型（Qwen-Plus）进行诊断 → 返回结构化诊断报告。

**诊断报告包含：**
- **错误标签** — 如「类型错误」「空指针」
- **根本原因** — 一句话概括
- **解决方案** — 分步骤说明
- **修正代码** — 可直接使用的修复后代码
- **详细分析** — Markdown 格式的调用栈分析和预防措施

### 📋 历史记录 — 过往记录回顾

以时间轴形式展示所有课堂摘要和代码诊断的历史记录，支持分页加载、展开详情和删除操作。

## 关键设计决策

### 超时保护策略

三层超时保护，防止用户长时间等待：

| 层级 | 超时值 | 说明 |
|------|--------|------|
| 客户端 | 55s | `wx.cloud.callFunction` 回调超时，强制解除 Loading |
| 云函数整体 | 50s | 云函数内部 `Promise.race` 整体超时 |
| HTTP 请求 | 30s | 单次 LLM API 请求超时 |

### 自定义 Markdown 解析器

`utils/markdown.js` 是一个纯自研的轻量级 Markdown 解析器，专为小程序 `rich-text` 组件设计。之所以不引入第三方库，是因为：

- 小程序对包体积敏感，第三方 Markdown 库体积较大
- 只需支持基础语法（标题、粗体、斜体、代码块、列表、引用、链接、分割线）
- 直接输出 `rich-text` 组件所需的 `nodes` 数组格式，无需中间转换

### 设计语言

采用 **"Codex · Midnight Library"（午夜图书馆）** 编辑风格：

- **暖色纸张色系** — 文字 `#e0dac8`，背景 `#1a1714`
- **黄铜金点缀** — 用于强调、分割线和选中态
- **封蜡红** — 用于代码诊断相关模块
- **纯 CSS 图标** — 十字准星、菱形、方形、圆形等符号均用 CSS 绘制，无 Emoji
- **环境光影** — 不对称径向渐变营造温暖的阅读氛围

## 环境变量

云函数通过 `config.json` 的 `env` 字段注入环境变量：

| 变量 | cloudfunctions/analyzeImage | cloudfunctions/diagnoseError |
|------|----------------------------|------------------------------|
| `LLM_API_URL` | DashScope API 地址 | 同上 |
| `LLM_API_KEY` | API 密钥 | 同上 |
| `LLM_MODEL` | `qwen-vl-max` | `qwen-plus` |

## 开始使用

### 前置条件

1. [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html) 已安装
2. 已注册微信小程序并开通**云开发**服务
3. 已在云开发控制台创建 `history` 数据库集合
4. 拥有阿里云 DashScope API Key

### 本地运行

1. **克隆项目并导入开发者工具**

   使用微信开发者工具打开 `learnproject` 目录，填入你的 AppID。

2. **配置云开发环境**

   在 `app.js` 中将 `env` 修改为你自己的云环境 ID：
   ```js
   wx.cloud.init({ env: '你的云环境ID', traceUser: true })
   ```

3. **配置 LLM API**

   修改每个云函数的 `config.json`，填入你的 DashScope API Key：
   ```json
   {
     "env": {
       "LLM_API_KEY": "sk-你的密钥"
     }
   }
   ```

4. **部署云函数**

   在微信开发者工具中，右键每个云函数文件夹 →「上传并部署：云端安装依赖」。

5. **创建数据库集合**

   在云开发控制台 → 数据库 → 新建集合 `history`，权限设置为「仅创建者可读写」。

6. **编译运行**

   点击开发者工具工具栏的「编译」按钮即可预览。
